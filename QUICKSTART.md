# 🚀 دليل البدء السريع - نظام رحلات مطار مسقط

## ⚡ التثبيت والتشغيل في 3 خطوات

### 1️⃣ تثبيت المتطلبات
```bash
pip install -r requirements.txt
```

### 2️⃣ تشغيل الإعداد التلقائي
```bash
python setup.py
```

### 3️⃣ البدء في جمع البيانات
```bash
# تشغيل مرة واحدة
python flight_manager.py

# أو تشغيل تلقائي مستمر
python auto_update.py
```

---

## 📁 ما الذي ستحصل عليه؟

بعد التشغيل الأول، ستجد مجلد `flight_data/` يحتوي على:

```
flight_data/
├── 📄 live_flights.json          ← الرحلات خلال 24 ساعة القادمة
├── 📄 archive_flights.json       ← أرشيف كامل (جميع الرحلات بدون تكرار)
├── 📊 live_flights.xlsx          ← نفس البيانات بصيغة Excel
├── 📊 archive_flights.xlsx       ← الأرشيف بصيغة Excel
└── ℹ️  metadata.json              ← معلومات عن آخر تحديث
```

---

## 🎯 الفرق بين الملفين الرئيسيين

### 📄 live_flights.json (ملف مؤقت)
- يحتوي على الرحلات التي **تقلع خلال 24 ساعة فقط**
- يتم **استبداله بالكامل** في كل تحديث
- مثالي للمراقبة الفورية

### 📚 archive_flights.json (فهرس دائم)
- يحتوي على **جميع الرحلات** منذ بداية التشغيل
- **لا يسمح بالتكرار**: كل رحلة تظهر مرة واحدة فقط
- **تحديث ذكي**: 
  - رحلة جديدة → تضاف
  - رحلة موجودة + تغيير في البيانات → تحديث فقط
  - رحلة موجودة + لا تغيير → تبقى كما هي
- يحتفظ بتاريخ **أول ظهور** و**آخر تحديث** لكل رحلة

## 🔑 مثال على آلية العمل

### اليوم الأول - 9 مايو الساعة 10 صباحاً
```json
// تم جلب رحلة جديدة
{
  "code": "WY101",
  "destination": "LHR",
  "status": "Scheduled",
  "firstSeen": "2026-05-09T10:00:00Z",  ← أول ظهور
  "lastUpdated": "2026-05-09T10:00:00Z"
}
```

### نفس اليوم - الساعة 2 ظهراً
```json
// نفس الرحلة لكن تغيرت الحالة
{
  "code": "WY101",
  "destination": "LHR",
  "status": "Boarding",                  ← تحدثت!
  "firstSeen": "2026-05-09T10:00:00Z",  ← لم يتغير
  "lastUpdated": "2026-05-09T14:00:00Z" ← تحدث!
}
```

### نفس اليوم - الساعة 3 مساءً
```json
// نفس البيانات بالضبط - لا تغيير
{
  "code": "WY101",
  "destination": "LHR",
  "status": "Boarding",                  ← نفس الحالة
  "firstSeen": "2026-05-09T10:00:00Z",  ← نفسه
  "lastUpdated": "2026-05-09T14:00:00Z" ← نفسه (لم يتحدث)
}
```

## 💡 نصائح مهمة

1. **التشغيل الأول قد يأخذ دقيقة**
   - النظام يجرب عدة طرق للوصول للموقع
   - هذا طبيعي بسبب حماية Cloudflare

2. **الرحلات القديمة لا تُحذف**
   - الأرشيف يحتفظ بكل شيء
   - يمكنك تحليل بيانات شهور أو سنوات

3. **التحديثات الذكية**
   - فقط البيانات المتغيرة يتم تحديثها
   - لا إهدار للمساحة أو التكرار

4. **اجمع البيانات لفترة طويلة**
   - كلما طالت فترة الجمع، كانت التحليلات أدق
   - ستكتشف أنماط مثيرة (أوقات الذروة، التأخيرات، إلخ)
